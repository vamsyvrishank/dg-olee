If a brute force solution uses two nested loops over the **same array**, and the inner loop doesn't restart from 0 each time — there's likely a two-pointer solution that gets it to O(n).



Cycle or gap between pointers needed? 
    → Fast/Slow

Sorted array, find pair?
    → Opposite Ends

Contiguous subarray/substring?
    → Sliding Window

In-place filter/partition?
    → Write Pointer
